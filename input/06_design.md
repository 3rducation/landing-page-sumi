# 06 Design Guide

## Design Goal
日系BtoB SaaSらしい分かりやすさと、学校・教育機関に受け入れられる信頼感を両立した、白ベースの1ページLPを作る。

## Keywords
- Japanese B2B SaaS
- School-friendly
- Trustworthy
- Clear
- Calm
- Professional
- Clean
- Warm
- Slightly premium
- Easy to understand

## Logo-Based Color Direction
The attached logo uses green and gold. Use white as the main background and apply green/gold as brand accents.

## Color Palette

### Base
- White: #FFFFFF
- Off White: #FAFAF7
- Light Gray: #F6F8F7
- Border Gray: #E5E7EB
- Text Main: #111827
- Text Sub: #4B5563

### Green
- Brand Green: #1F6B4D
- Deep Green: #0F3D2E
- Fresh Green: #2F9E73
- Pale Green: #EAF6F0
- Mist Green: #F3FAF6

### Gold
- Brand Gold: #A88B3D
- Light Gold: #E9D8A6
- Pale Gold: #FFF8E1

### Accent
- Navy: #0F172A
- Slate: #334155
- Blue Gray: #EAF2F8

## Color Usage
- Use white / off-white for most backgrounds.
- Use green for primary CTA, logo accents, section labels, and icons.
- Use gold sparingly for premium accents, underlines, dividers, or stat highlights.
- Do not make the whole page green.
- Avoid neon or cyberpunk AI aesthetics.
- Avoid childlike school colors.

## Typography
Use Japanese-friendly system font stack:
- system-ui
- -apple-system
- BlinkMacSystemFont
- "Hiragino Sans"
- "Yu Gothic"
- "Noto Sans JP"
- sans-serif

## Typography Rules
- Body line-height: 1.7 to 1.9
- Headings: bold, clear, but not too heavy
- Japanese paragraphs should be readable with enough spacing
- Avoid very small text

## Layout
- LP max width: 1120px to 1200px
- Section padding desktop: 88px to 120px
- Section padding mobile: 56px to 80px
- Use large, clear section headings
- Use cards and step layouts
- Use visual mockups generously

## Header
- Sticky header
- White / translucent white background
- Logo on left
- Anchor links on right
- Primary CTA button
- Mobile menu can be simple or stacked

## Hero
- White or very pale green background
- Left: headline, subcopy, CTA
- Right: dashboard / product mockup
- Add stat card: 採点時間 約75%削減
- Add small trust badges
- Use subtle gold line or glow behind hero elements

## Visual Direction
Use a mix of:
- Attached logo
- SaaS dashboard-style mockups
- Answer sheet preview
- Score table
- Class analytics chart
- Data flow diagram style
- Workflow cards
- Feature icons

## Image Rules
- Use `public/images/logo-transparent.png` for the logo.
- Use proposal PDF screenshots only as reference or temporary visual assets.
- If using proposal page images, present them inside clean modern cards, not as raw slides.
- If screenshots are too rough, create HTML/CSS mockups that look like product UI.
- Do not use random stock photos unless necessary.

## Card Style
- Background: white
- Border: #E5E7EB
- Border radius: 20px to 28px
- Shadow: subtle
- Padding: 24px to 32px
- Hover: small shadow/translate only

## Buttons

### Primary
- Background: Brand Green
- Text: White
- Border-radius: 9999px or 14px
- Hover: Deep Green

### Secondary
- Background: White
- Text: Deep Green
- Border: Pale Green or Light Gold
- Hover: Mist Green

## Sections
Each section should include:
- Eyebrow label
- Heading
- Short lead copy
- Visual or cards

## Motion
- Minimal fade-in and slide-up only
- No excessive animation
- No aggressive parallax
- No flashy AI effects

## Accessibility
- Semantic HTML
- Alt text for meaningful images
- Keyboard-accessible links/buttons
- Visible focus states
- High contrast
- Do not communicate important information by color only

## Overall Impression
A school administrator should feel:
- This company understands school operations
- This product is useful and practical
- The AI supports teachers, not replaces them
- The design is modern but not risky
- It is worth requesting a demo
